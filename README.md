# [艾森豪矩陣任務優先權決策引擎]

## Proposal Report

### 動機與目標
在現代快節奏的生活與工作環境中，無論是面對多重報告的學生，還是處理繁雜專案的職場人士，常會面臨「任務超載」的困境。傳統的條列式待辦清單（To-Do List）僅能記錄事項，卻無法客觀衡量任務間的相對權重。這導致人們極易陷入「急迫性偏見（Urgency Bias）」——將大量精力耗費在眼前緊急卻不重要的瑣事上，反而一再推遲真正對長期目標與成長有益的關鍵任務。

目標為開發一個基於「艾森豪矩陣」理論的 CLI 終端機工具，透過量化「重要性」與「緊急性」兩個維度，自動為任務計算優先權分數，並利用高效的資料結構確保在任務繁多時仍能快速排定執行順序，協助使用者做出理性的時間管理決策。

### 預期功能
- 任務量化輸入： 使用者可新增任務，並為其「重要性」與「緊急性」進行評分（1-10 分）。
- 自動化優先權排序： 系統根據加權公式（例如：重要性占 40%，緊急性占 60%）自動計算權重，並即時維護一個「當前最應執行」的任務清單。
- 四象限視覺化輸出： 在終端機以 ASCII 圖表形式，將任務分類顯示在「重要且緊急」、「重要不緊急」、「不重要但緊急」、「不重要不緊急」四個區塊。
- 動態狀態管理： 支援更新任務狀態（完成、修改權重、刪除），並確保排序結果同步更新。
- 資料持久化： 支援將任務清單匯出與匯入為 CSV 或 JSON 格式，確保資料不會因程式關閉而遺失。

### 使用技術
- 程式語言： Python 。
- 開發環境： VS Code / Git (GitHub)。
- 外部套件： prettytable 或 tabulate (用於終端機表格美化)。

### 時程規劃
- Week 7 (Proposal)： 完成資料結構設計與核心權重演算法定義。
- Week 8-9： 實作基礎 CRUD（增刪查改）功能與核心排序模型。
- Week 10-11 (Prototype)： 完成核心 Heap 排序邏輯，達成能在終端機輸出排序後清單的雛形。
- Week 12-13： 實作四象限視覺化圖表輸出與檔案讀寫功能。
- Week 14： 進行壓力測試（模擬上百筆任務時的效能）與 Bug 修復。
- Week 15 (Final)： 錄製 Demo 影片並完成最終報告手稿。

### 與課程的關聯
1. Priority Queue / Max-Heap (優先權佇列 / 最大堆積)：  
   這是本專題的核心，我們不需要對所有任務進行全排序 $O(N \log N)$，我們只需要隨時知道「最高優先權」的任務是什麼。
   使用 Max-Heap 可以讓取出最高優先權任務的時間複雜度達到 $O(1)$，新增任務也僅需 $O(\log N)$。這在處理大量動態變動的任務時，效能遠優於一般的 Array。
3. Hash Table (雜湊表 / Python Dictionary)：  
   為了實現任務的快速查找，當使用者想要修改某個特定任務（例如：將「期末專題」標記為完成）時，透過 Hash Table 可以將任務名稱或 ID 直接對應到其在記憶體中的位址，達到 $O(1)$ 的查詢效率，避免線性搜尋。
4. Weighted Scoring Algorithm (加權評分演算法)：  
   透過這門課學到的邏輯思考，設計一個合理的公式來平衡「重要性」與「緊急性」。例如針對接近 Deadline 的任務給予非線性的權重加成。
6. 2D Array / List of Lists (二維陣列)：  
   在進行「四象限」視覺化分類輸出時，使用二維空間的概念來儲存各象限的任務清單，方便程式進行區域性的巡訪與列印。
